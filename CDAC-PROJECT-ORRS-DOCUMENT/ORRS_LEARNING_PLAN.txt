@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) {
        http.csrf().disable()
            .authorizeHttpRequests()
            .requestMatchers("/api/users/login", "/api/users/register").permitAll()
            .anyRequest().authenticated();
        return http.build();
    }
}

DAY 19-20: DISTRIBUTED TRANSACTIONS (SAGA PATTERN)
---------------------------------------------------
Learn:
- Saga pattern (choreography)
- Compensating transactions
- Event-driven architecture basics

Implement:
‚úÖ Booking flow: Lock seats ‚Üí Create booking ‚Üí Payment ‚Üí Ticket
‚úÖ Rollback logic if any step fails
‚úÖ Test failure scenarios

Code Example:
public class BookingSaga {
    public BookingResult execute(BookingRequest request) {
        try {
            seatService.lockSeats(request.getSeats());
            Booking booking = bookingService.create(request);
            Payment payment = paymentService.process(booking);
            Ticket ticket = ticketService.generate(booking);
            return new BookingResult(booking, ticket);
        } catch (Exception e) {
            rollback(); // Unlock seats, cancel booking
            throw e;
        }
    }
}
